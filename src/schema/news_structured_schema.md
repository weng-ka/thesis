# 勞工新聞結構化特徵 Schema

本 Schema 用於對 `data/news_dataset/raw` 中的每篇勞工新聞進行結構化抽取，輸出至 `data/news_dataset/structured`。

- `metadata.date`、`metadata.source`、`metadata.author` 由腳本從源站元數據解析
- 其餘所有欄位由 LLM 抽取

---

## JSON 結構定義

```jsonc
{
  "metadata": {
    "title": "",                    // string（不可為空）：新聞標題
    "author": "",                   // string（不可為空）：作者或機構；源站無則填 "NA"
    "date": "",                     // string（不可為空，YYYY-MM-DD）：發布日期
    "source": "",                   // string（不可為空）：新聞來源網址（原文 URL）
    "identifier": ""                // string（不可為空）：唯一標識符，格式 "NEWS-{檔名編號}"（如 "NEWS-0221"）
  },

  "5W1H": {
    "who": "",                      // string（不可為空）：涉及的勞動者、用人單位及相關主體
    "what": "",                     // string（不可為空）：發生了什麼勞動事件
    "when": "",                     // string（不可為空）：事件發生時間或時間段
    "where": "",                    // string（不可為空）：事件發生地點
    "why": "",                      // string（不可為空）：事件發生原因
    "how": ""                       // string（不可為空）：事件經過及處理方式
  },

  // 勞工議題分類（不可為空；可多選）
  // 可選值：勞動合同與勞動關係 | 勞動爭議 | 工資、報酬與最低工資 |
  //         工作時間與加班制度 | 休息休假與帶薪年休假制度 |
  //         解除勞動合同、裁員、經濟補償與賠償 |
  //         職業病、醫療期與患病勞動者保護 | 工傷保險與工傷認定 |
  //         社會保險與福利制度 | 勞務派遣與非典型用工 |
  //         女職工與未成年工特殊保護 | 就業促進與勞動保障監察
  "themes": [],

  "events": [                       // 陣列：每一新聞事件為一個物件
    {
      "date": {
        "event_date_start": "",     // string（YYYY-MM-DD，可為空）：單日則只填 start
        "event_date_end": ""        // string（YYYY-MM-DD，可為空）
      },

      // 事件發生省份（不可為空；僅選其一）
      // 可選值：省級行政區名稱（如 北京市、陝西省、湖南省、廣東省 等）
      "province": "",

      // 工人職業身份（不可為空；可多選）
      // 可選值：白領受僱者 | 藍領受僱者 | 政府公務員或事業單位工作者 |
      //         攤販/店主/小業主 | 青年學生/職校/實習生 | 不明
      "worker_identity": [],

      // 工人年齡分布（不可為空），未提及則填 ["不明"]
      "worker_age": [],

      // 資方類型（不可為空；可多選）
      // 可選值：國企/央企 | 外資 | 政府/事業單位 | 民營企業 |
      //         平台企業 | 勞務公司/派遣機構 | 不明
      "company_identity": [],

      // 涉及行業（不可為空；可多選）
      // 可選值：採礦業 | 製造業 | 交通物流業 | 建築業 |
      //         服務業 | 黨政機關 | 農業 | 其他
      "involved_industry": [],

      // 工人話語：陣列，每一工人個體為一個物件；若無可辨識個體則為空陣列 []
      "worker_discourse": [
        {
          "name": "",                    // string（不可為空）：工人個體稱呼
          "gender": "",                  // string（不可為空）：從稱呼推理，"男" / "女" / "不明"
          "personal_statements": []      // string[]（可為空陣列）：該工人的所有話語或引述，每句一個字串
        }
      ],

      "worker_situation": {
        // 工人所受傷害（不可為空；可多選）
        // 可選值：死亡 | 身體傷害 | 健康權損害 | 經濟損害 |
        //         精神或人格尊嚴傷害 | 職業發展損害 |
        //         社會關係損害 | 人身自由損害 | 無具體傷害 | 其他
        // 區分說明：
        //   - 身體傷害：已發生的工傷、疾病、殘疾等實際身體損害
        //   - 健康權損害：健康權利被剝奪但傷害尚未顯現（如未體檢、職業病防護缺失、未繳醫保等）
        "harm_experienced": [],
        "rights_violated": [],      // string[]（可為空陣列）：從事實提煉的核心權益問題，每個問題一個字串
        "responses_to_violations": [
          {
            "entity": "",            // string（不可為空）：回應實體（如「地方勞動局」「醫院副院長」等）
            "response": ""           // string（不可為空）：該實體的具體回應內容或處置措施
          }
        ],
        "compensation_received": "" // string（可為空）：工人實際獲得的補償
      }
    }
  ]
}
```

---

## 事件拆分規則

- 若主體或時間/地點明顯不同，則拆分為不同事件
- 同一事件的後續處置及回應，視為同事件的子信息，不另立事件
